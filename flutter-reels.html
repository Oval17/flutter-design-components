TIP · 01
Use const
constructors
Mark widgets as const to skip rebuilds and ship 10× faster screens.
// ❌ rebuilds every time
Text('Hello')

// ✅ cached forever
const Text('Hello')
save it 👉 share with a Flutter friend
TIP · 02
Lazy-load with
.builder
ListView() renders ALL children up-front. ListView.builder() only builds what's visible. Huge perf win on long lists.
ListView.builder(
  itemCount: items.length,
  itemBuilder: (ctx, i) =>
    ListTile(title: Text(items[i])),
)
smooth scrolling at any size 🚀
TIP · 03
Hot reload vs
hot restart
Different keys, very different speed. Use the right one.
⚡ Hot Reload
Press r
Keeps app state
UI tweaks only
~1s
🔄 Hot Restart
Press R
Resets state
main() reruns
~5s
90% of the time, you just want lowercase r
TIP · 04
Use Slivers for
custom scrolls
Need a collapsing header? Mixed grid + list? Sticky sections? Slivers handle all of it in one scroll view.
CustomScrollView(
  slivers: [
    SliverAppBar(pinned: true),
    SliverList(...),
    SliverGrid(...),
  ],
)
⚡ buttery smooth complex scrolls
TIP · 05
Riverpod in
3 lines
State management without context, with compile-safe access. The Flutter team's recommendation.
final counter =
  StateProvider<int>((ref) => 0);

// in widget
final count = ref.watch(counter);
no Provider.of, no listen=false 🙏
TIP · 06
Return multiple
values with records
Dart 3 records let you return two things from a function — no class, no tuple package, no fuss.
(int, String) getUser() {
  return (42, 'Anurag');
}

final (id, name) = getUser();
destructuring in Dart 🎯
TIP · 07
Custom colors,
type-safe
ThemeExtension lets you add brand colors that Theme.of() exposes directly — no magic strings, no hacks.
class BrandColors extends
  ThemeExtension<BrandColors> {
  final Color success;
  final Color warning;
  // + copyWith + lerp
}
no more if(isDark) ternaries 🎨
